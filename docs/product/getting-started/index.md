## What first launch looks like

Once Sentinel opens, the flow is simple:

1. add or select a workspace
2. start a thread
3. pick an engine, model, and mode
4. start working

If the workspace points at a real local repo, the app gets much more useful.

## Main screens

- `/` starts from the new thread flow
- `/thread/[threadId]` is the main thread screen
- `/automations` is for recurring runs
- `/skills` is for installable skills
- `/settings/*` is where the setup lives

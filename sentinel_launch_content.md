## LinkedIn Announcement Post

Sentinel is open source today.

It's a desktop app for working with AI on your codebase. You open a folder, start a thread, and from there the whole loop is in one place — talking through the problem, writing code, running it, reviewing what changed, making a commit, opening a PR. The built-in terminal and browser stay attached to the same thread, so you're not constantly switching tabs and losing context.

I started building it after a few months of using OpenAI's Codex app. The workflow clicked for me. But I wanted to use other models depending on the task, and I wanted everything to stay on my machine. Sentinel does both.

It supports 14+ providers: Anthropic, OpenAI, Google, Mistral, Ollama, Groq, Amazon Bedrock, xAI, GitHub Copilot, and more. You pick the model. Everything else stays the same.

A few other things it handles: plan mode for tasks that need more than a quick exchange, memory so the app remembers context across sessions, recurring automations for things you run regularly, voice input, file and document attachments, and approvals so nothing runs without you seeing it first.

It also connects to external tools if you want — GitHub, Linear, Notion, Slack, Google Drive, and a handful of databases. Everything stays local by default though. Nothing leaves your machine unless you configure it to.

Still alpha. Some parts are solid, some I'm still working through. I use it every day.

Hi, I'm Mohamed Achaq.

A few months ago I was using the Codex app — OpenAI's coding tool. The workflow made sense to me. Open a folder, start a thread, work through something, see what changed, run commands. I kept coming back to it.

But I wanted to use other models. Claude, Gemini, local models. Whatever felt right for the task. And I wanted everything to stay on my machine. So I built Sentinel.

It's a desktop app. You open a folder, create a thread, and start working. The thread can be a regular chat or a plan — plan mode is useful when the task is bigger and you want to map it out before you start making changes.

While you work, the terminal and browser are built in and stay close to that thread. So reviewing a diff, running your tests, checking something in the browser — it's all in the same window. When you're ready, you commit, push, and open a PR without leaving the app.

You pick the model. Sentinel supports over 14 providers — Anthropic, OpenAI, Google, Mistral, Ollama, Groq, Amazon Bedrock, xAI, GitHub Copilot, and more. Swap the model whenever you want. The rest of the app stays exactly the same.

There's also memory, so the app can carry context across sessions without you having to re-explain the project every time. Automations for recurring tasks. Voice input if you'd rather talk than type. Approvals so you stay in control of what actually runs. And connections to tools like GitHub, Linear, Slack, Notion, Google Drive, and a few databases, if you want to bring those into the same workflow.

Everything runs locally. Your data stays on your machine. Nothing goes out unless you set up an external connection yourself.

Today it's open source. Builds for macOS, Windows, and Linux are on GitHub Releases.

It's still alpha — some parts are solid, some I'm still working through. I use it every day.

The repo is github.com/Cronacl/Sentinel.

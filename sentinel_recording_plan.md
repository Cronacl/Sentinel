# Sentinel — Single-Take Recording Plan

Record everything in one continuous session. Pause 2–3 seconds between sections — that pause becomes your cut point in editing.

---

## Before you start

- Clean up the app: close any messy threads, have a tidy workspace ready
- Pick a project folder that looks good (short name, recognisable)
- Pre-type any prompts you'll use so you're not typing live (copy-paste is fine)
- Hide your dock if you can — cleaner recording
- Set resolution to your native display, record the whole screen or just the app window

---

## Recording sequence

**1. Open the app**
Start with the app already open on the workspace list. Let it sit for 2 seconds. This is your establishing shot.

**[PAUSE 3s]**

**2. Open a workspace**
Click into a workspace. Let the sidebar load with threads visible. Don't click anything yet — just let it sit for 3 seconds so the UI is readable.

**[PAUSE 3s]**

**3. Start a new thread**
Click "New thread". Type a prompt — something short and visual like:

> "Add a search bar to the header"

Hit send. Let the model start responding and making file edits. Watch 2–3 edits come through. Stop before it finishes.

**[PAUSE 3s]**

**4. Switch to plan mode**
Open a new thread or switch mode. Type something that benefits from planning:

> "Refactor the authentication flow"

Show the plan being generated — the list of steps appearing. Let 3–4 steps load.

**[PAUSE 3s]**

**5. Show the diff**
Either let the previous task finish, or click into an existing thread that already has file changes. Open the diff panel. Let the green/red lines show for 3–4 seconds. Scroll slightly so it looks live.

**[PAUSE 3s]**

**6. Commit**
From the diff view, click commit. Fill in a commit message if needed (or let it auto-fill). Show the commit going through. Don't push — just commit is enough.

**[PAUSE 3s]**

**7. Open the terminal**
Open the bottom terminal panel. Type and run something short:

> `npm run dev` or `bun dev`

Let it start. Show a line or two of output. You don't need it to fully boot.

**[PAUSE 3s]**

**8. Open the built-in browser**
Switch to the browser panel. Navigate to localhost or wherever the app runs. Let the page load. Sit on it for 3 seconds.

**[PAUSE 3s]**

**9. Open the model picker**
Click the model dropdown. Let the full list sit open for 3–4 seconds — don't rush. The list of providers is the whole point. Then pick a different model and close.

**[PAUSE 3s]**

**10. Show integrations / memory / automations**
Go to whichever one looks cleanest in your app — integrations list, memory panel, or automations. Let it sit open for 4 seconds. If it's the integrations list, scroll slowly so names are readable (GitHub, Slack, Notion, etc.).

**[PAUSE 3s]**

**11. End shot**
Go back to the main workspace view — threads visible, sidebar showing. Let it sit for 3 seconds. This is your closing frame before the title card.

---

## After recording

In editing, use the pauses to find your cut points. Each section between pauses becomes one clip. Drop a text card before or after each clip based on the storyboard.

You don't need every section to be perfect. If one clip looks off, you can either re-record just that section or cut it from the video entirely — the storyboard has enough clips that you can drop one or two and still have a complete video.
